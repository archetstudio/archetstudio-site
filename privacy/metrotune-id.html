Archet Studio LLC (selanjutnya disebut “Kami”) berkomitmen untuk melindungi Data Pribadi Anda sesuai dengan Undang-Undang Nomor 27 Tahun 2022 tentang Pelindungan Data Pribadi (“UU PDP”). Kebijakan ini menjelaskan cara Kami mengumpulkan, memproses, menyimpan, dan melindungi Data Pribadi Anda ketika Anda menggunakan aplikasi seluler MetroTune™ dan situs web Kami. Apabila Anda memiliki pertanyaan atau kekhawatiran mengenai kebijakan ini atau praktik pelindungan data Kami, silakan hubungi Kami sebagaimana tercantum dalam Bagian 14 (Pengaduan dan Kontak).

Ringkasan

  • Mikrofon: Audio diproses sepenuhnya di perangkat secara real-time. Tidak pernah direkam, disimpan, atau ditransmisikan.
  • Iklan: Kami menggunakan Google AdMob (sebagai jaringan iklan utama dan mediasi), Pangle (ByteDance), Vungle/Liftoff, PubMatic, serta OpenX sebagai sumber permintaan iklan berbasis penawaran (bidding), dan Firebase Analytics (Google) yang terhubung dengan AdMob untuk analitik penggunaan aplikasi dan kinerja iklan (permintaan, tayangan, interaksi, serta perkiraan pendapatan iklan per wilayah, sesi, versi, dan kohort — pelaporan monetisasi) untuk mendukung biaya operasional aplikasi. Pada iOS, MetroTune™ telah mengimplementasikan kerangka App Tracking Transparency (ATT); IDFA hanya tersedia bagi SDK iklan apabila Anda memberikan izin pelacakan melalui permintaan ATT. Apabila ATT ditolak atau belum diberikan, iklan ditampilkan dalam bentuk non-personalisasi. Pada Android, Anda mengontrol personalisasi melalui pengaturan persetujuan. Unity Ads dapat ditambahkan sebagai mitra mediasi AdMob pada rilis mendatang.
  • Penjualan data: Kami tidak pernah menjual Data Pribadi Anda.

2. Data Pribadi yang Dikumpulkan dan Cara Pengumpulan

Kami mengumpulkan Data Pribadi bersifat umum sebagai berikut. Kami tidak mengumpulkan Data Pribadi bersifat spesifik (sensitif) sebagaimana dimaksud dalam Pasal 4 ayat (2) UU PDP.

2.1. Melalui Korespondensi Email

  • Kategori data: nama, alamat email, isi pertanyaan, serta informasi teknis yang Anda berikan secara sukarela (jenis perangkat, versi sistem operasi).
  • Cara pengumpulan: Anda memberikan informasi ini secara sukarela melalui email — inquiry@archetstudio.com (pertanyaan umum), support@archetstudio.com (dukungan aplikasi), privacy@archetstudio.com (permintaan terkait Data Pribadi).
  • Informasi teknis yang Anda berikan secara sukarela saat menghubungi dukungan hanya digunakan untuk menyelesaikan masalah teknis dan meningkatkan produk Kami. Kami tidak menjual informasi tersebut atau menggunakannya untuk tujuan pemasaran.

2.2. Pengumpulan Otomatis melalui SDK Iklan Pihak Ketiga

MetroTune™ mengintegrasikan SDK iklan pihak ketiga berikut untuk tujuan pengiriman dan penawaran iklan (ad bidding):

  • Google Mobile Ads SDK (AdMob) — pengiriman iklan utama dan mediasi.
  • Pangle (ByteDance) — permintaan iklan berbasis penawaran.
  • Vungle / Liftoff — permintaan iklan berbasis penawaran.
  • PubMatic — permintaan iklan berbasis penawaran melalui mediasi AdMob.
  • OpenX — permintaan iklan berbasis penawaran melalui mediasi AdMob.
  • Firebase Analytics (Google) — analitik penggunaan aplikasi dan kinerja iklan; terhubung dengan AdMob untuk pelaporan monetisasi per wilayah, sesi, versi, dan kohort.

Akan datang: Unity Ads dapat ditambahkan sebagai mitra mediasi AdMob pada rilis mendatang. Saat diintegrasikan, Unity Ads akan tunduk pada praktik data yang sama sebagaimana dijelaskan dalam bagian ini, serta pada kebijakan privasi Unity sendiri (https://unity.com/legal/privacy-policy).

MetroTune™ sendiri tidak mengumpulkan atau mengunggah Pengenal Iklan secara langsung. Setiap jaringan iklan dapat mengumpulkan kategori data berikut ketika iklan diminta atau ditampilkan:

  • Pengenal iklan — AAID (Android Advertising ID, aktif secara default pada Android). Pada iOS, IDFA hanya tersedia bagi SDK iklan apabila Anda memberikan izin pelacakan melalui permintaan App Tracking Transparency (ATT) Apple; apabila ATT ditolak atau belum diberikan, iOS sendiri menyetel IDFA menjadi nol pada tingkat sistem operasi, dan MetroTune™ juga menyetel requestNonPersonalizedAdsOnly = true pada setiap permintaan iklan sehingga iklan ditayangkan secara non-personalisasi. IDFV (Identifier for Vendor, pengidentifikasi khusus aplikasi pada iOS, tidak memerlukan persetujuan pengguna berdasarkan kerangka ATT) dapat dikirim ke jaringan mediasi iklan untuk tujuan atribusi iklan dan pencegahan penipuan terlepas dari status ATT.
  • Alamat IP (lokasi perkiraan berbasis IP).
  • Data interaksi iklan (tayangan, klik, penyelesaian iklan berhadiah).
  • Data teknis SDK (tidak ada SDK analitik, pelaporan kerusakan, atau jejaring sosial terpisah yang diintegrasikan).

Data tersebut hanya dikirim ke jaringan iklan ketika iklan diminta atau ditampilkan, dan digunakan untuk tujuan atribusi iklan serta pencegahan penipuan. Kami tidak mengoperasikan basis data sisi server atas data pengguna; Kami hanya meneruskan data yang dikumpulkan SDK selama proses pengiriman iklan.

2.3. Audio Mikrofon (Pemrosesan Sementara)

  • MetroTune™ menggunakan mikrofon untuk deteksi nada (tuner) dan deteksi tepuk tangan ganda untuk penentuan tempo (metronom).
  • Audio diproses sepenuhnya di perangkat — tidak pernah direkam, disimpan, atau ditransmisikan.
  • Tidak dilakukan pengenalan suara.

2.4. Penyimpanan Lokal di Perangkat (Tidak Dikirim ke Server)

  • Pengaturan pengguna (tema warna, status buka kunci premium).
  • Stempel waktu penyelesaian iklan berhadiah.

Data penyimpanan lokal tidak meninggalkan perangkat Anda dan tidak dikirim ke server mana pun.

3. Tujuan dan Dasar Hukum Pemrosesan

Kami memproses Data Pribadi Anda untuk tujuan-tujuan berikut berdasarkan dasar hukum sebagaimana diatur dalam UU PDP:

  • Menanggapi pertanyaan dan memberikan dukungan melalui email: Pemenuhan perjanjian — diperlukan untuk memenuhi permintaan Anda sebelum atau selama pemberian layanan (Pasal 20 ayat (2) huruf b).
  • Menampilkan iklan non-personalisasi: Kepentingan sah — untuk mendukung biaya operasional aplikasi gratis (Pasal 20 ayat (2) huruf f).
  • Menampilkan iklan yang dipersonalisasi: Persetujuan — Anda dapat memberikan atau menarik persetujuan kapan saja (Pasal 20 ayat (2) huruf a).
  • Pemrosesan audio mikrofon (tuner/metronom): Persetujuan — Anda memberikan izin melalui pengaturan izin sistem operasi (Pasal 20 ayat (2) huruf a).
  • Penyimpanan data di perangkat: Pemenuhan perjanjian — diperlukan untuk menyediakan fitur aplikasi (Pasal 20 ayat (2) huruf b).

Dalam hal Kami mengandalkan persetujuan sebagai dasar hukum, Anda dapat menarik persetujuan tersebut kapan saja tanpa memengaruhi keabsahan pemrosesan yang telah dilakukan sebelum penarikan persetujuan. Untuk informasi lebih lanjut mengenai cara menarik persetujuan, lihat Bagian 9.

6. Jangka Waktu Penyimpanan Data

Data yang Kami kumpulkan akan disimpan selama diperlukan untuk memenuhi tujuan sebagaimana ditetapkan, kecuali peraturan perundang-undangan mengatur lain:

  • Korespondensi dan dukungan email: 12 bulan setelah penyelesaian, kemudian dihapus secara permanen.
  • Penayangan iklan (SDK pihak ketiga): Sesuai kebijakan penyimpanan data masing-masing jaringan iklan dan analitik — Google, Pangle, Vungle, PubMatic, OpenX, Firebase.
  • Data aplikasi lokal di perangkat: Dihapus otomatis saat aplikasi di-uninstall.

Setelah jangka waktu penyimpanan berakhir atau tujuan pemrosesan telah tercapai, Kami akan memusnahkan Data Pribadi sesuai dengan Pasal 47 UU PDP. Harap diperhatikan bahwa data yang dikumpulkan oleh pihak ketiga (seperti Google, Pangle, Vungle/Liftoff, PubMatic, OpenX, dan Firebase Analytics) mungkin memiliki jangka waktu penyimpanan yang berbeda sesuai kebijakan privasi masing-masing pihak ketiga tersebut.

7. Langkah-Langkah Keamanan

Kami menerapkan langkah-langkah teknis dan organisasional yang memadai untuk melindungi Data Pribadi Anda sebagaimana diwajibkan oleh Pasal 35 UU PDP, yang meliputi:

  • Langkah organisasional: Akses terhadap Data Pribadi dibatasi hanya kepada personel yang memerlukan.
  • Langkah teknis: Seluruh koneksi jaringan (termasuk AdMob, Pangle, Vungle, PubMatic, OpenX, dan Firebase Analytics; Unity Ads apabila ditambahkan pada rilis mendatang) dienkripsi melalui HTTPS/TLS. Kami tidak mengoperasikan server backend atau penyimpanan cloud sendiri.
  • Minimalisasi data: Kami hanya mengumpulkan data yang secara wajar diperlukan untuk tujuan yang telah ditetapkan. MetroTune™ sendiri tidak mengumpulkan, menyimpan, atau mengunggah pengenal pengguna (IDFA, AAID, IDFV) secara langsung; penanganan pengenal hanya terjadi di dalam SDK iklan yang terintegrasi ketika iklan diminta atau ditampilkan.
  • Privacy Manifest (iOS): File PrivacyInfo.xcprivacy tingkat aplikasi menyatakan NSPrivacyTracking = true. Kunci NSPrivacyTrackingDomains sengaja tidak dicantumkan pada tingkat aplikasi; pengungkapan domain pelacakan didelegasikan kepada manifes privasi (privacy manifest) masing-masing SDK iklan yang terintegrasi, yang dikumpulkan oleh Apple ke dalam binari pada saat build.
  • Penyimpanan lokal: Pengaturan pengguna hanya disimpan di perangkat Anda dan tidak dikirim ke server eksternal mana pun.

8. Pemberitahuan Pelanggaran Data Pribadi

Sesuai dengan Pasal 46 UU PDP, dalam hal terjadi kegagalan pelindungan Data Pribadi, Kami wajib menyampaikan pemberitahuan secara tertulis kepada Subjek Data Pribadi dan Lembaga Pelindungan Data Pribadi paling lambat 3 × 24 jam (tiga kali dua puluh empat jam) sejak diketahuinya kegagalan pelindungan tersebut.

Pemberitahuan tersebut paling sedikit memuat:

  • Data Pribadi yang terungkap;
  • Kapan dan bagaimana Data Pribadi terungkap;
  • Upaya penanganan dan pemulihan atas kegagalan pelindungan tersebut oleh Kami.

9. Pengenal Iklan dan Cara Menolak

Aplikasi MetroTune™ menggunakan Google Mobile Ads SDK (AdMob), Pangle (ByteDance), Vungle/Liftoff, PubMatic, OpenX, dan Firebase Analytics SDK, yang dapat mengumpulkan pengenal iklan (AAID pada Android) dan alamat IP secara otomatis. Pada iOS, MetroTune™ telah mengimplementasikan kerangka App Tracking Transparency (ATT) Apple. Permintaan ATT ditampilkan pada saat Anda pertama kali mencoba menonton iklan video berhadiah; pilihan Anda kemudian disimpan dalam cache dan digunakan untuk percobaan berikutnya. Apabila Anda memberikan izin ATT, IDFA dapat tersedia bagi SDK iklan untuk tujuan atribusi iklan, personalisasi, dan pencegahan penipuan. Apabila Anda menolak ATT atau belum diminta, iOS sendiri menyetel IDFA menjadi nol pada tingkat sistem operasi, dan MetroTune™ juga menyetel requestNonPersonalizedAdsOnly = true pada setiap permintaan iklan, sehingga iklan ditayangkan secara non-personalisasi. IDFV (pengidentifikasi khusus aplikasi pada iOS, tidak memerlukan persetujuan pengguna berdasarkan kerangka ATT) dapat dikirim ke jaringan mediasi iklan untuk tujuan atribusi iklan dan pencegahan penipuan terlepas dari status ATT. MetroTune™ sendiri tidak mengumpulkan IDFA, AAID, atau IDFV secara langsung; penanganan pengenal hanya terjadi di dalam SDK iklan ketika iklan diminta atau ditampilkan. Unity Ads dapat ditambahkan sebagai mitra mediasi AdMob pada rilis mendatang dan akan tunduk pada logika yang sama. Situs web Kami (archetstudio.com) tidak menggunakan cookie atau teknologi pelacakan serupa.

Pilihan Anda — cara menonaktifkan pengumpulan otomatis:

  • iOS: Pengaturan > Privasi & Keamanan > Pelacakan → matikan “Izinkan Aplikasi Meminta Melacak”, atau tolak permintaan ATT saat MetroTune™ menampilkannya. Apabila ATT tidak diberikan, seluruh iklan ditampilkan secara non-personalisasi.
  • Android: Pengaturan > Google > Iklan > Pilih keluar dari Personalisasi Iklan.
  • EU/EEA: Formulir persetujuan Google UMP ditampilkan saat peluncuran pertama. Untuk mengubah pilihan persetujuan, hubungi privacy@archetstudio.com.

Menonaktifkan pengumpulan tidak memengaruhi kemampuan Anda menggunakan aplikasi. Iklan akan tetap ditampilkan dalam bentuk non-personalisasi.

Akses Internet (iOS dan Android): SDK iklan dan analitik yang disebutkan (Google AdMob, Pangle, Vungle/Liftoff, PubMatic, OpenX, Firebase Analytics; Unity Ads apabila ditambahkan pada rilis mendatang) menggunakan koneksi Internet untuk pengiriman iklan. Fungsi ini beroperasi berdasarkan prinsip upaya terbaik — jika tidak ada koneksi jaringan, iklan tidak akan ditampilkan, tetapi seluruh fitur lain dari aplikasi tetap berfungsi secara normal.

10. Hak Subjek Data Pribadi

Sesuai dengan UU PDP, Anda sebagai Subjek Data Pribadi memiliki hak-hak berikut:

  • Hak untuk mendapatkan informasi (Pasal 5): Mendapatkan informasi tentang kejelasan identitas, dasar kepentingan hukum, tujuan permintaan dan penggunaan Data Pribadi, serta akuntabilitas pihak yang meminta Data Pribadi.
  • Hak untuk melengkapi, memperbarui, dan/atau memperbaiki (Pasal 6): Melengkapi, memperbarui, dan/atau memperbaiki kesalahan dan/atau ketidakakuratan Data Pribadi.
  • Hak untuk mengakses (Pasal 7): Mendapatkan akses dan memperoleh salinan Data Pribadi.
  • Hak untuk mengakhiri pemrosesan, menghapus, dan/atau memusnahkan (Pasal 8): Mengakhiri pemrosesan, menghapus, dan/atau memusnahkan Data Pribadi apabila telah melampaui masa retensi atau tidak lagi diperlukan untuk tujuan pemrosesan.
  • Hak untuk menarik persetujuan (Pasal 9): Menarik kembali persetujuan pemrosesan Data Pribadi yang telah diberikan. Penarikan persetujuan tidak memengaruhi keabsahan pemrosesan yang telah dilakukan sebelumnya.
  • Hak untuk mengajukan keberatan (Pasal 10): Mengajukan keberatan atas tindakan pengambilan keputusan yang hanya didasarkan pada pemrosesan secara otomatis.
  • Hak untuk menunda atau membatasi pemrosesan (Pasal 11): Menunda atau membatasi pemrosesan Data Pribadi secara proporsional.
  • Hak untuk menggugat dan menerima ganti rugi (Pasal 12): Menggugat dan menerima ganti rugi atas pelanggaran pemrosesan Data Pribadi sesuai dengan ketentuan peraturan perundang-undangan.
